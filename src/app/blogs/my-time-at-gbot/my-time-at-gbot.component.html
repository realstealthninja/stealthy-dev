They introduced us to Arduino, A single board micro-controller. basically a programmable PCB. An arduino works on variant of C. I prefer to call it embeded C but that's probably Very wrong. There are 14 digital pins and 6 analog pins. Whats the difference? A digital pin is digital meaning it can input or output a fixed value in the form of a square wave; Either 0 or 1 (on or off or high or low). While an analog pin can input a wide range of values in the form of a sine wave think of it was values from 0 to 1 (eg: 0.34). An arduino is a digital board and cant directly input or ouput analog values thus it must covert it to digital in the case of input or convert from digital to analog in the case of output. Luckily the board's chip the ATmega328P has builtin functionality of converting to and from analog. This results values to be displayed in 10 bits that is 0-1024

What does a arduino file look like?

Good question! An arduino is coded using a file with an extension of .ino usually named something like sketch⟨somenumber⟩.ino which has two parts a setup function and a loop function. The setup function runs the code inside only ONCE and the loop function as you've guessed it runs it in a loop until the end of time or until you power the arduino off.

If you look carefully you'll see a tiny LED on the 13th pin. You might be wondering how we even begin to light it. Well it starts off by setting how the pin should operate. Since a LED is usually either on or off we connect it to a Digital pin. Now we must inform the board that we are trying to OUTPUT a signal to pin 13. Hence we must set the mode of the pin to "OUTPUT" using the built in function called pinMode

To send an ON signal to that pin we use the builtin function called "digitalWrite" in our loop to turn it and make it stay on forever. To blink a given led you must add a delay. a delay works as follows, it asks the board to wait a while before executing the other code. For example if you wanted to turn off the LED you might try to add which would blink the LED yes but it would be invisible as it is in a loop it blinks at microsecond intervals basically per cpu clock. To avoid this we use a delay so the code gets a chance to be executed and waits a bit before executing the other parts. For ease of access i have moved this to a tiny function
